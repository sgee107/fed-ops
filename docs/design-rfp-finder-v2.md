# Design: RFP Finder v2 — Profile-Driven Agentic Search

**Date:** 2026-05-03
**Status:** Approved for planning

---

## 1. Goal

Evolve the current single-input search page into a profile-driven agentic RFP discovery tool that demonstrates:
- Exa's full API surface (search, findSimilar, answer/research)
- Claude-powered agentic tool use with streaming
- System prompt context injection per company profile
- Offline agentic data generation (profile simulation)

No persistent storage. All state is ephemeral per session.

---

## 2. What We Are Building

A Next.js web app where a user selects one of three fictional company profiles, triggers an AI agent to find relevant federal RFPs, refines results via chat, and deep-dives individual opportunities via a streaming research drawer — all powered by Exa and Claude.

The app doubles as a pitch artifact: it demonstrates to procurement/BD teams what a real profile-aware RFP monitoring tool would feel like, with a banner making clear this is an example.

---

## 3. Company Profiles (Simulated, Generated Offline)

Three industry archetypes, differentiated by size, sector, and certification posture. Profiles are generated by a one-time offline agent script and committed as static JSON.

| Profile | Employees | Sector | Key Certs | Contract Sweet Spot |
|---|---|---|---|---|
| **Apex Defense Solutions** | ~110 | Defense IT / Cybersecurity | CMMC L2, cleared staff | DoD IT modernization, $500K–$5M |
| **MedBridge Health IT** | ~40 | Healthcare IT | 8(a), WOSB | HHS, VA, DoD Health Agency, $100K–$2M |
| **Cascade Infrastructure** | ~260 | Civil construction & environmental | HUBZone | Army Corps, DoT, EPA, $1M–$20M |

Size differences are intentional: small firms should surface set-aside and SBIR opportunities; large firms target full-and-open competitions. This demonstrates profile-aware search targeting.

## 4. Offline Profile Generation Script

### Purpose

Rather than hand-writing fictional profiles, an offline agent script uses Claude + Exa to research real companies in each archetype space, then synthesizes realistic fictional profiles from findings. This demonstrates an agentic offline workflow pattern.

### Invocation

```bash
npm run create-sim-profiles
```

### Script: `scripts/create-sim-profiles.ts`

Run via `tsx`. Uses `generateText` (non-streaming, fire-and-forget) with:
- **Claude** as the LLM
- **Exa tools** to research real companies in each archetype
- Structured output (JSON) for the final profiles

**Agent loop:**
1. For each archetype seed, agent calls `webSearch` to find 3–5 real companies
2. Agent extracts: typical size, certifications, target agencies, NAICS codes, contract ranges
3. Agent synthesizes a fictional company inspired by findings
4. Script writes result to `src/data/profiles.json`

**Output shape:**

```typescript
interface CompanyProfile {
  id: string;                    // "apex-defense"
  name: string;                  // "Apex Defense Solutions"
  sector: string;                // "Defense IT / Cybersecurity"
  employees: number;
  certifications: string[];      // ["CMMC L2", "cleared personnel"]
  naicsCodes: string[];          // ["541512", "541519"]
  targetAgencies: string[];      // ["DoD", "USAF", "Army"]
  contractSizeMin: number;       // 500000
  contractSizeMax: number;       // 5000000
  description: string;           // 2–3 sentence company bio
  searchBias: string;            // appended to agent queries: "DoD IT modernization cybersecurity..."
  systemPromptContext: string;   // injected verbatim into agent system prompts
}
```

Script outputs `src/data/profiles.json` as `CompanyProfile[]`. Committed to the repo so the app never needs to run the script at runtime.

## 5. Application Architecture

### Tech Stack Additions

| Package | Purpose |
|---|---|
| `ai` (Vercel AI SDK) | `streamText`, `generateText`, `useChat` |
| `@ai-sdk/anthropic` | Claude provider |
| `@exalabs/ai-sdk` | Pre-built `webSearch` tool for Vercel AI SDK |
| `zod` | Tool parameter schemas (for custom tools) |
| `tsx` | Run TypeScript scripts offline |

`@exalabs/ai-sdk` exposes a single export — `webSearch()` — which is a fully-formed Vercel AI SDK tool. It reads `EXA_API_KEY` from env automatically. `findSimilar` and `researchOpportunity` are written as custom tools wrapping `exa-js` directly.

### Exa Tools (shared across agents)

```typescript
// src/lib/exa-tools.ts

import { webSearch } from '@exalabs/ai-sdk';
import { tool } from 'ai';
import { z } from 'zod';
import Exa from 'exa-js';

const exa = new Exa(process.env.EXA_API_KEY!);

// Pre-built tool from @exalabs/ai-sdk — handles search with domain filtering
export const rfpWebSearch = webSearch({
  includeDomains: ['sam.gov', 'grants.gov', 'usaspending.gov', 'fpds.gov', 'acquisition.gov'],
  numResults: 8,
  contents: {
    highlights: true,
    summary: true,
    text: { maxCharacters: 2000 },
    livecrawl: 'preferred',
  },
});

// Custom tool — exa-js findSimilar (findSimilarAndContents is deprecated)
export const findSimilar = tool({
  description: 'Find similar RFPs and opportunities to a given URL',
  parameters: z.object({ url: z.string() }),
  execute: ({ url }) => exa.findSimilar(url, {
    numResults: 5,
    contents: { highlights: true, summary: true },
  }),
});

// Custom tool — exa-js answer endpoint for deep research synthesis
export const researchOpportunity = tool({
  description: 'Research a specific opportunity in depth — synthesizes deadline, budget, requirements, and set-aside type from multiple sources',
  parameters: z.object({ question: z.string() }),
  execute: ({ question }) => exa.answer(question),
  // Returns: { answer: string, citations: SearchResult[] }
});
```

## 6. API Routes

### `POST /api/find-rfps`

**Input:** `{ profileId: string }`

**Behavior:** Streaming agent that:
1. Injects selected company's `systemPromptContext` into system prompt
2. Constructs 2–3 profile-targeted queries using `searchBias` and NAICS codes
3. Calls `webSearch` multiple times (max 5 tool steps)
4. Deduplicates and returns prioritized RFP results

**Stream:** `streamText` response — client renders results as they arrive via `useChat` or direct stream parsing.

**System prompt template:**
```
You are an RFP sourcing agent for {name}, a {employees}-person {sector} firm.
{systemPromptContext}

Your job: find the most relevant federal contracting opportunities for this company.
Construct targeted search queries using the company's NAICS codes, certifications, and agency relationships.
Run multiple searches to maximize coverage. Prioritize opportunities that match contract size range ${contractSizeMin}–${contractSizeMax}.
Return results in a structured format.
```

---

### `POST /api/chat`

**Input:** `{ messages: Message[], profileId: string, rfpContext: RFPResult[] }`

**Behavior:** Conversational agent that:
1. Has full context of current RFP results in system prompt
2. Can call `webSearch` to find additional opportunities based on user direction
3. Has guardrail: refuses off-topic requests

**System prompt template:**
```
You are an RFP research assistant for {name}.
{systemPromptContext}

Current search results in context:
{rfpContext — title, url, highlights for each result}

GUARDRAIL: Only answer questions related to government contracting, federal procurement, RFPs, solicitations, and related business development topics. If asked about anything unrelated, politely decline and redirect to procurement topics.
```

**Max tool steps:** 3

---

### `POST /api/research`

**Input:** `{ rfp: RFPResult, profileId: string }`

**Behavior:** Streaming research agent that:
1. Calls `researchOpportunity` to get Exa-synthesized answer on deadline, budget, NAICS, set-aside type, and key requirements
2. Calls `findSimilar` to surface related opportunities
3. Synthesizes a fit assessment for the selected company profile
4. Streams everything live — client shows tool call indicators as agent works

**System prompt:**
```
You are a government contracting research analyst. Research the following RFP for {name}.

Use the researchOpportunity tool to answer: What is the submission deadline, estimated budget/ceiling, set-aside type, primary NAICS code, and key technical requirements for this opportunity?

Then use the findSimilar tool to find 3–5 related opportunities.

Finally, provide a brief fit assessment: how well does this opportunity match {name}'s capabilities and certifications?
```

**Max tool steps:** 4

## 8. Agentic Primitives Demonstrated

| Primitive | Where |
|---|---|
| **Tool calling** | All three API routes — agent decides when/which Exa tool to invoke |
| **Multi-step planning** | `/api/find-rfps` — agent runs 2–3 searches, deduplicates, prioritizes |
| **Streaming with tool indicators** | `/api/research` — drawer shows "Researching opportunity..." then "Finding similar..." live |
| **System prompt context injection** | Every route — profile data baked into system prompt |
| **Guardrailed agent** | `/api/chat` — system prompt enforces topic scope |
| **Offline agentic workflow** | `create-sim-profiles` — agent-generated static data, separate from the app |
| **Parallel-capable tool calls** | `/api/research` — `answer` + `findSimilar` can be parallelized |

## 10. Out of Scope (Next Phase)

- Competitor intel (who has won similar contracts)
- Persistence / saved searches
- Real company profile ingestion (SAM.gov entity lookup)
- Notifications / scheduled re-runs
- RFP document parsing / full text analysis
- Filtering UI (date range, contract value slider, agency filter)

## 11. Open Questions / Risks

| Item | Status | Notes |
|---|---|---|
| `exa.answer()` availability | **Resolved** | Confirmed in `exa-js` type definitions. Supports both sync and streaming variants. Returns `{ answer: string, citations: SearchResult[] }`. |
| `findSimilarAndContents` deprecation | **Resolved** | Use `exa.findSimilar(url, { contents: {...} })` — the combined method is deprecated in favor of this signature. |
| `findSimilar` domain filtering | **Accepted risk** | `findSimilar` does not expose `includeDomains`. Results may include non-federal URLs. Claude's synthesis step in `/api/research` should note when a similar result is off-domain. |
| Latency of `/api/find-rfps` | **Open** | Multi-step agent with 3+ Exa calls could be slow. Streaming mitigates perceived latency but worth measuring in implementation. |
| `create-sim-profiles` API key requirements | **Open** | Script needs both `ANTHROPIC_API_KEY` and `EXA_API_KEY` at generation time. Add a note to README. |
